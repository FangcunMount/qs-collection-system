# 问卷填写页面测试指南

## 测试环境准备

### 1. 数据准备

需要准备以下测试数据：

- **受试者数据**：至少创建一个受试者
- **问卷数据**：创建一个包含多种题型的测试问卷
- **用户登录**：确保已登录并获取 Token

### 2. 测试用例

#### 测试用例 1：正常流程测试

**步骤**：
1. 打开小程序
2. 选择或创建受试者
3. 选择一个问卷
4. 查看信息确认页面
5. 点击"确认信息，开始填写"
6. 填写所有必填题
7. 点击"提交问卷"
8. 验证跳转到答卷分析页面

**预期结果**：
- 信息确认页面正确显示问卷和受试者信息
- 问卷题目正确渲染，题号正确
- 提交成功后跳转到分析页面

#### 测试用例 2：必填验证测试

**步骤**：
1. 进入问卷填写页面
2. 跳过必填题，直接点击提交
3. 验证错误提示

**预期结果**：
- 显示"第X题为必填题"的提示
- 页面自动滚动到未填写的题目

#### 测试用例 3：字数验证测试

**步骤**：
1. 找到有字数限制的文本题
2. 输入少于最小字数的内容
3. 点击提交
4. 输入超过最大字数的内容
5. 点击提交

**预期结果**：
- 提示"最少字数为X"
- 提示"最大字数为X"

#### 测试用例 4：数值验证测试

**步骤**：
1. 找到数字题
2. 输入小于最小值的数字
3. 点击提交
4. 输入大于最大值的数字
5. 点击提交

**预期结果**：
- 提示"最小值为X"
- 提示"最大值为X"

#### 测试用例 5：多选题验证测试

**步骤**：
1. 找到多选题
2. 选择少于最小选择数的选项
3. 点击提交
4. 选择超过最大选择数的选项
5. 点击提交

**预期结果**：
- 提示"最少选择X个选项"
- 提示"最多选择X个选项"

#### 测试用例 6：多受试者选择测试

**步骤**：
1. 创建多个受试者
2. 进入问卷填写页面
3. 选择其中一个受试者

**预期结果**：
- 显示受试者选择弹窗
- 选择后正确加载该受试者信息

#### 测试用例 7：题目显示隐藏测试

**步骤**：
1. 找到有显示条件的题目
2. 选择不同的选项
3. 观察关联题目的显示/隐藏

**预期结果**：
- 符合条件时题目显示
- 不符合条件时题目隐藏
- 隐藏的题目不参与提交

## API 测试

### 1. 获取问卷详情

```bash
curl -X GET "http://localhost:18083/api/v1/questionnaires/{code}" \
  -H "Authorization: Bearer {token}"
```

**验证点**：
- 返回问卷基本信息
- 返回所有题目
- validation_rules 是数组格式
- 每个 rule 有 rule_type 和 target_value

### 2. 获取受试者详情

```bash
curl -X GET "http://localhost:18083/api/v1/testees/{id}" \
  -H "Authorization: Bearer {token}"
```

**验证点**：
- 返回受试者基本信息
- 包含姓名、性别、出生日期等

### 3. 提交答卷

```bash
curl -X POST "http://localhost:18083/api/v1/answersheets" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "questionnaire_code": "test_code",
    "questionnaire_version": "1.0",
    "testee_id": 1,
    "title": "测试答卷",
    "answers": [
      {
        "question_code": "q1",
        "question_type": "Radio",
        "value": "option1",
        "score": 5
      }
    ]
  }'
```

**验证点**：
- 返回答卷 ID
- 返回成功消息

## 常见问题排查

### 问题 1：加载失败

**症状**：页面一直显示 loading

**排查**：
1. 检查 Token 是否有效
2. 检查问卷 code 是否正确
3. 检查受试者 ID 是否正确
4. 查看网络请求响应

### 问题 2：提交失败

**症状**：点击提交后报错

**排查**：
1. 检查是否所有必填题已填写
2. 检查数据格式是否正确
3. 查看控制台错误信息
4. 检查 API 返回的错误信息

### 问题 3：题目不显示

**症状**：某些题目不显示

**排查**：
1. 检查题目的 show_controller 配置
2. 检查关联题目的答案是否符合条件
3. 查看 getQuestionIsShow 函数的返回值

### 问题 4：验证规则不生效

**症状**：验证规则没有触发

**排查**：
1. 检查 validation_rules 格式是否正确
2. 检查 rule_type 和 target_value 是否匹配
3. 查看 checkQuestion 函数的执行

## 性能测试

### 测试指标

- **页面加载时间**：从进入到显示信息确认页面
- **题目渲染时间**：从确认到显示所有题目
- **提交响应时间**：从点击提交到跳转

### 测试场景

1. **小问卷**：10题以内
2. **中等问卷**：10-50题
3. **大问卷**：50题以上

### 预期性能

- 页面加载 < 2秒
- 题目渲染 < 1秒
- 提交响应 < 3秒
