# UI Baseline

本目录记录全链路前端重构的微信开发者工具截图基线。截图只用于评审，不参与小程序构建。

## Device Matrix

| Key | Viewport | Purpose |
| --- | --- | --- |
| `small` | 320 × 568 | 检查小屏、固定底栏和长文本拥挤 |
| `standard` | 375 × 812 | 主要视觉验收基线 |
| `large` | 430 × 932 | 检查大屏留白和卡片密度 |

## Scenario Matrix

| Area | Required states |
| --- | --- |
| 首页 | 已登录有报告、无报告、机构任务入口 |
| 目录 | 加载完成、空数据、长标题、分类筛选 |
| 作答 | 测评准备、单档案、多档案、单题、多题、校验失败 |
| 记录 | 首次加载、空数据、已出报告、处理中、失败、筛选面板 |
| 报告 | 医学报告、人格报告、缺少可选段落、等待、失败 |
| 我的 | 未登录、已登录、档案列表、表单校验错误 |

## Capture Rules

1. 使用 Node 16 构建最新 `dist`，再由微信开发者工具打开项目。
2. 每次只更新当前重构批次涉及的截图。
3. 文件名使用 `<area>--<state>--<device>.png`，例如 `records--empty--small.png`。
4. 截图前固定测试数据和系统字体大小；不得使用设计稿代替运行时截图。
5. 对比时重点检查导航胶囊避让、内容溢出、触控区域、固定底栏、安全区、颜色和字重。
6. 任何有意视觉变化都必须在批次说明中记录；无法解释的差异视为回归。
